About the Project

EdgeBot MK-II is a minimalist, yet powerful robot platform designed as Zimbabwe's official open-source robotics standard. It operates on ESP32 using MicroPython and is controlled via Wi-Fi and a mobile-friendly web interface.

1. Motor Setup

We configure the 2 motors (A & B) using Pin and PWM:

# Motor A
in1 = Pin(18, Pin.OUT)
in2 = Pin(19, Pin.OUT)
ena = PWM(Pin(25), freq=1000)

# Motor B
in3 = Pin(26, Pin.OUT)
in4 = Pin(27, Pin.OUT)
enb = PWM(Pin(14), freq=1000)

Pin Layout: GPIOs 18/19 and 26/27 control motor direction, while 25 and 14 control speed (PWM).

Pro Tip: The L298N driver has 2 channels, but each channel can power 2 motors if connected in parallel. That means you can control 4 motors (2 synced per channel). Perfect for 4WD builds!

3. Wi-Fi Access Point

The ESP32 becomes its own Wi-Fi hotspot:

ap = network.WLAN(network.AP_IF)
ap.active(True)
ap.config(essid="EdgeBot-MKII", password="edgebot123", authmode=3)

WPA2 secured network that users can connect to directly.

4. HTML Web Interface

The control console is built right into the code. When a button is clicked, it sends a command:

<button class="btn" name="action" value="forward">FORWARD</button>

Works on any browser: phone, tablet, or PC!

5. Web Server

Sets up a simple socket server to listen for incoming connections on port 80:

addr = socket.getaddrinfo('0.0.0.0', 80)[0][-1]
s = socket.socket()
s.bind(addr)
s.listen(1)

Every time a client connects and presses a button, the corresponding movement function is triggered.

6. USB Serial Command Interface

Connect to the robot via USB and use commands:

LIST            #  List files on the device
READ filename   #  Read file contents
ID              #  Display device identity

7. Debugging & Developer Tips

  • Power Tips: Separate motor & logic power for stability. Use capacitors near motor inputs (100Î¼F+).
  • Test GPIOs: Blink an LED or use serial print()s to verify logic is working before attaching motors.
  • Slow Response? Limit connections to 1 controller. Use a controller_ip lock to filter requests.
  • Boot Safely: Always call stop() on boot to prevent movement at startup.
  • Extendability: Add endpoints like /status to show live robot info (IP, motor state, RAM).
